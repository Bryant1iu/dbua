# 项目重构说明

## 技术栈
Python 3.x

## 常用命令
- 运行测试：pytest（或 python -m pytest）
- 代码风格检查：flake8 . 或 ruff check .
- 格式化：black . 或 autopep8

## 重构目标
1. 拆分超过 100 行的函数，每个函数只做一件事
2. 统一代码风格（遵循 PEP 8：4 空格缩进、snake_case 命名）
3. 消除重复代码，提取公共函数到 utils.py 或对应模块
4. 改善命名：变量/函数名要见名知意，避免 data、tmp、x 等
5. 为复杂逻辑补充 docstring 和行内注释

## 硬性约束
- 不改变任何对外暴露的函数签名和接口
- 不删除现有功能，只做内部整理
- 每完成一个文件必须 git commit 一次
- 重构后逻辑必须与原来完全一致
